# 涡旋模块：OW → YOLO 伪标签开发参考

> **目的**：在命题方 NetCDF 仅提供 `adt` / `ugos` / `vgos` 格点场、**不含像素级标注**的前提下，为「物理先验生成伪标签 → YOLO-seg 训练」提供可落地的技术路线与文献锚点。  
> **数据依据**：`服创数据集/数据集说明.md`（META4.0 DT 思路、IoU、气旋/反气旋两类、二维场作多通道输入）；实文件结构见 `docs/命题方数据集说明.md` 与中尺度涡 `eddy_dataset --inspect` 记录。

**与命题方沟通**：若官方另行提供 IoU 真值或评测脚本，本文档中的伪标签策略需与评测口径对齐后再冻结版本。

---

## 1. 总体策略（建议实施顺序）

1. **路径五（物理混合）**：先提高「候选涡旋区域」的**初始纯度**，减少后续过滤负担。  
2. **路径二（不确定性 / 集成）**：对多配置 OW（及可选 WA、VG、SSH 等值线）结果做**一致性投票**与置信度分层。  
3. **路径一（双向恢复）**：在时间维上关联同一涡旋实例，**恢复**单帧 OW 弱响应导致的低质量多边形。  
4. **路径三（质量解耦修正）**：区分「中心/位置可信」与「边界不完整」，对后者做**参考修正**而非整例丢弃。  
5. **路径四（非对称阈值与共现）**：在 CE/AE 两类与空间邻域上细化**正负与共现**规则（注意：文献中的 ADTCR「ADT」与海洋 **ADT 动力高度** 缩写无关，避免混写）。  
6. **迭代自训练 + 动态阈值**（见 §3）：在 YOLO 有一定能力后滚动扩充伪标签，并按海域/季节做小批量锚点校准 OW 阈值。

---

## 2. 五条可选开发路径（等级与适配）

以下 **等级** 表示在本项目中的推荐优先级：**S** 强烈建议纳入首版管线，**A** 首版可简化实现或二期，**B** 增强项。

### 路径一：双向伪标签恢复（Bi-Directional Pseudo Label Recovery）

| 项目 | 内容 |
|------|------|
| **核心思想** | 传统伪标签常丢弃低置信度样本；本思路利用**前向 + 后向**目标跟踪，同时使用过去与未来信息，恢复被丢掉的低置信度标签，并与高置信度融合得到精炼伪标签。 |
| **代表性文献** | Lee, S., et al. *Improving Object Detection Accuracy with Self-Training Based on Bi-Directional Pseudo Label Recovery.* **Electronics**, 2024, 13(12), 2230.（YOLOv5 验证；BDD100K mAP@50 +0.52%；Waymo 10% 标注场景 mAP@50 约 8.1% → 8.7%–9.9%） |
| **对 OW→YOLO 的适配** | 涡旋在 **daily 序列**上具有轨迹连续性：将 OW 连通域 + 多边形在时间上做**前后向关联**；若某帧 OW 弱导致多边形退化或置信度低，可用相邻帧稳定多边形**插值/传播**恢复该帧伪标签。 |
| **建议等级** | **A**（首版可做简化版：仅前向或仅 IoU/质心匹配追踪） |

### 路径二：不确定性估计与置信度筛选

| 项目 | 内容 |
|------|------|
| **核心思想** | 单一 softmax 置信度往往乐观；通过**多模型集成、熵、贝叶斯**等得到更可靠的不确定性，再用**动态阈值/分位数**筛选伪标签。 |
| **关键技术点** | 熵采样 + 动态分位数阈值；集成/多头方差作样本权重；教师-学生下**像素级不确定性图**引导训练。 |
| **对 OW→YOLO 的适配** | 并行多组 **OW 参数**（及可选不同平滑核、不同 SSH 阈值）得到多张 W 场；**多配置同时响应**的区域赋予更高伪标签置信度，优先入训练集；冲突区域降权或剔除。 |
| **建议等级** | **S**（实现成本相对可控，与路径五可合并为第一里程碑） |

### 路径三：伪标签质量解耦与修正

| 项目 | 内容 |
|------|------|
| **核心思想** | 将**检测质量**与**分割/边界质量**分两阶段评估：保留「位置对、边界差」的样本并修正，而非整例丢弃。 |
| **代表性文献** | Lu, Y., et al. *Pseudo-Label Quality Decoupling and Correction for Semi-Supervised Instance Segmentation.*（COCO 实例分割；准确率 +2.3%，约恢复 76% 低质量预测） |
| **对 OW→YOLO 的适配** | OW 常产生**不完整闭合**多边形：先判**涡心/质心**是否落在高可信区域（检测阶段），再单独评**边界闭合度/弧长比**；若检测高、边界低，可用邻域高置信实例的边界形态或凸包/D-P 简化结果做**保守修正**。 |
| **建议等级** | **A** |

### 路径四：非对称双重阈值与标签共现关系

| 项目 | 内容 |
|------|------|
| **核心思想** | 多标签稀疏场景下，对**负标签用较低阈值、正标签用较高阈值**；利用**标签共现**从弱响应区挖掘潜在正标签。（文献来自医疗多标签分类 ADTCR 等，**ADTCR 中的 ADT ≠ 海洋 ADT**） |
| **对 OW→YOLO 的适配** | **气旋（CE）/反气旋（AE）** 在空间上可能相邻或嵌套：若 AE 高置信，其邻近弱响应区可**提高**对 CE 候选的敏感度（或反之），需在物理上可解释的前提下**保守**使用，避免假阳性激增。 |
| **建议等级** | **B**（首版可只做 CE/AE 分阈值，不做复杂共现） |

### 路径五：物理参数混合集成（Hybrid Detection, HD）

| 项目 | 内容 |
|------|------|
| **核心思想** | 单独 OW 阈值敏感、假阳/假阴随海区变化；**混合检测（HD）** 将 OW 与 **SSH/SLA 等值线** 等几何约束结合，并可识别**多核**结构。 |
| **代表性文献** | *Enhancing the accuracy of automatic eddy detection and the capability of recognizing the multi-core structures from maps of sea level anomaly.*（HD 与 SLA 等值线思路）  
| **扩展** | OW 与 **Winding-Angle（WA）、Vector Geometry（VG）** 等集成可提升鲁棒性。 |
| **对 OW→YOLO 的适配** | 由 **adt** 派生 SLA/SSH 相关场，与 **ugos/vgos** 计算 OW；**多物理方法投票**一致区域才生成伪标签，作为**高质量种子**集。 |
| **建议等级** | **S**（与命题方「仅三变量、不可擅自加字段」一致：不新增 NetCDF 变量，仅在内存中计算派生量） |

## 3. 完整 OW → YOLO 管道的关键环节

| 环节 | 内容 | 备注 |
|------|------|------|
| **连通域 → 多边形** | 不规则连通域可先 **凸包** 得粗边界，**Douglas–Peucker** 简化折线，必要时用 **最小外接矩形** 得 YOLO 检测框；分割支路优先保留**归一化多边形**（YOLO-seg）。 | 矩形会损失形状信息，分割头可缓解 |
| **类别** | `class_id`：气旋 / 反气旋（与数据集说明一致）；OW 符号与类型映射规则需在代码与文档中**固定**。 | |
| **标注格式** | YOLO：`class` + 检测框或 **segment 多边形**（归一化到 0–1）。 | 与 `data/processed/eddy/dataset.yaml` 一致 |
| **迭代自训练** | 伪标签训练 YOLO → 对未标注帧推理 → **高置信度**预测并入伪标签池 → 再训练；循环需**版本化**与验证集**绝不混入**训练池。 | 赛题要求验证集不用于训练 |
| **阈值策略** | OW 对阈值极度敏感；可用**分区/分季节**动态阈值，或用少量**锚点样本**（若命题方允许）反标定局部阈值。 | 与路径二、四结合 |

## 4. 主要挑战与限制

| 挑战 | 描述 |
|------|------|
| OW 根本局限 | 单一 W 阈值难以全球统一，海区/季节漂移大。 |
| 噪声传递 | 伪标签噪声会直接教坏 YOLO；必须配合路径二、三、五的**过滤与修正**。 |
| 分布漂移 | 黑潮延伸体内部也有时空差异；需分块评估伪标签质量或域自适应。 |
| 多边形→框的信息损失 | 外接矩形弱化边界；**优先 YOLO-seg + 多边形伪标签**。 |
| 评测对齐 | 最终 IoU 若依命题方 mask，伪标签流水线需在文档中说明**与真值的差异及风险**。 |

## 5. 参考路线图（阶段 ↔ 方法 ↔ 文献）

| 管道阶段 | 可参考方法 | 文献 / 实现提示 |
|----------|------------|-----------------|
| OW 计算 + 多物理集成 | 混合检测 HD；OW + SSH/SLA 等值线；可选 WA、VG | HD 论文；META/Copernicus 产品说明 |
| 连通域 → 多边形 | 凸包 + Douglas–Peucker | 计算几何标准算法 |
| 伪标签置信度 | 多阈值 OW 集成；熵/分位数动态阈值 | 路径二 |
| 低置信度恢复 | 时间维双向/单向跟踪融合 | Lee et al. 2024 |
| 部分正确样本 | 检测-分割质量解耦与修正 | Lu et al. |
| 共现与阈值 | 非对称阈值；CE/AE 邻域规则（慎用） | ADTCR 思路迁移（勿与海洋 ADT 混淆） |
| 迭代优化 | 自训练 + 动态阈值 | 半监督检测/分割通用流程 |

**实施建议（与上文 §1 一致）**：**路径五 → 路径二 → 路径一 → 路径三 →（可选）路径四**，全程穿插 **§3 的格式与自训练规范**。

## 6. 与仓库实现的对接点（已实现首版）

- **入口**：`python -m src.preprocess.eddy_dataset --export-yolo --data-config config/data.yaml [选项…]`  
  等价于 `python -m src.preprocess.eddy_yolo_export …`（参数见 `--help`）。  
- **物理**：`src/preprocess/eddy_physics.py` — 地转流梯度 → ζ 与 Okubo–Weiss `W`；**路径二**简化为多分位阈值掩膜投票（`--vote-percentiles` / `--vote-min`，或 `--single-percentile`）。  
- **输出**：`data/processed/eddy/images/{train,val,test}/`、`labels/{train,val,test}/`、`dataset.yaml`（`path: .` 相对该目录）；与 `config/eddy.yaml` 的 `paths.dataset_yaml` 默认一致。  
- **烟测**：建议加 `--max-frames-per-file 3 --time-stride 60` 控制体量。  
- **训练 / 评估**：`python -m src.eddy.train --config config/eddy.yaml`（非 smoke）；`python -m src.eddy.eval --config config/eddy.yaml --ckpt outputs/eddy/best.pt`。  
- **局限**：首版为 **OW+投票+连通域→多边形**，未实现路径一/三/四/五中的 HD、双向跟踪与质量解耦；类别由核区 ζ 符号 **启发式** 映射 CE/AE，**需与命题方 IoU 真值或 META 产品对齐后校准**。
