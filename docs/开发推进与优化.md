| 优先级 | 项 | 说明 |
|--------|----|------|
| P0 | **命题方划分 + 全量数据** | 仓库默认 **`config/data.yaml` → `hydro_year_split.enabled: true`**；预处理见下「正式对照命令」。 |

## 正式对照：全量 + 命题方年份划分

```bash
cd ~/autodl-tmp/EddyFusion   # 或本机仓库根
# 确认 config/data.yaml 中 hydro_year_split.enabled 为 true（默认已开）
python -m src.preprocess.hydro_dataset \
  --config config/hydro_hycom.yaml \
  --from-nc --data-config config/data.yaml \
  --year-split --stride 24

python -m src.hydro.train --config config/hydro_hycom.yaml

python -m src.hydro.eval --config config/hydro_hycom.yaml --ckpt outputs/hydro/best.pt --split val
python -m src.hydro.eval --config config/hydro_hycom.yaml --ckpt outputs/hydro/best.pt --split test
# 输出：outputs/hydro/metrics_summary_val.json 与 metrics_summary_test.json
```

若某划分无 nc 文件，预处理会跳过该 split；需保证 `服创数据集/海域要素预测` 下含训练年、2015（验证）、2014（测试）等数据。

### 预处理内存与 `Killed`

训练年日文件可达 **7000+**，当前实现会先把各 split **整段拼成 `(T,H,W,C)` 浮点场** 再滑窗；峰值还与 **滑窗样本数 N** 有关（`X+y` 约 `N×(T_in+T_out)×H×W×C×4` 字节）。实例内存不足时进程会被 Linux **OOM killer** 直接 `Killed`（与数据是否存在无关）。

- **`hydro_preprocess.max_train_daily_files`**：默认 **800**，仅截断训练排序后的日文件数。需更大训练覆盖面时在 **内存足够** 的机器上增大该值，或设为 **`null`** 尝试全量。
- **命题方划分**下预处理已改为 **按 train → val → test 顺序**：每个 split 在滑窗与 Z-score 后 **立即写出 npz 并释放张量**，避免原先「三个 split 的滑窗数组同时驻留」导致测试阶段再分配约 **20+ GiB** 时被 Kill。
- 可选 **`max_val_daily_files` / `max_test_daily_files`**：进一步截断验证/测试日文件（`null` 为不截断）。
- 预处理开始会打印 **预估拼接场 GiB**；若仍 OOM，可减小上述日文件上限或增大 `--stride` 以减小窗口数 N。
