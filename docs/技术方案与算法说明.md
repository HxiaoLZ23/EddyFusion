# 技术方案与算法说明

与《A09-最终技术方案与降级策略》及企业材料④对齐；**仓库当前实现**如下（无本地实验数字时以 `outputs/` 与评测脚本为准）。

## 水文（海域要素预测 · HYCOM）

- **任务**：由过去 **168** 个时间步（与命题方 hourly 口径对齐，见 `hydro_hycom.yaml`）预测未来 **72** 步（72 小时）。
- **输入输出张量**：样本形状 `X: (N, T_in, H, W, C)`、`y: (N, T_out, H, W, C)`，`C=4` 对应 **temp / sal / u / v**（映射自 SST、SSS、SSU、SSV），见 `config/variable_map.yaml`。
- **模型**：`HydroBaseline` — 多层 **ConvLSTM** 编码时序；**单卷积头**输出 `C×T_out` 再 reshape（避免 72 路独立解码占用显存）；可选 **通道 SE**；训练期可开 **`encoder_checkpoint`**（长序列减激活显存）。
- **损失与评测**：训练 **MSE**；评测脚本汇总 **RMSE、按通道 NRMSE**（与命题方「均方误差→NRMSE」口径对齐，见 `docs/命题方数据集说明.md`）。
- **预处理**：`src/preprocess/hydro_dataset.py` + `hydro_nc_stack.py` — 多日 NetCDF 按时间拼接 → 滑窗 → 训练集上 **Z-score**（`stats/hydro_zscore.npz`）→ `data/processed/hydro/*.npz`。
